Styling usings the class attribute

Using CSS you can provide styling for an element.

...
<style>p { color: black; }</style>
...
<p>This paragraph has black font</p>

The above code renders as:

This paragraph has black font

Notice that this paragraph, however, has different margins. That's because it has the class attribute set to "body".

I've defined the p.body style as:

 p.body {
                margin-left: 5em;
                margin-right: 5em;
            }

This means paragraphs p with the body class have left and right margins set to 5em.

When defining CSS, you can use the element name for a specific element, and then a dot/period/full stop, then the class name (x.y). This means any element x with class y uses the formatting.

You can also just define a class name using the dot notation (.classname). For example:

.red { color: red;}

So I can make any element red just by adding the class attribute.

You can also stack classes, so this paragraph has all the formatting from the p element, the body class, and the red class.

So you can do some very powerful things based on how you design all your styles, and how you combine them.

This text is red, this text is yellow, but what happens when you combine them. Does order matter?

The code for the above paragraph is:

<p class="body">This text is <span class="red">red</span>, this text is <span class="lemon">yellow</span>, but what happens when you
        <span class="red lemon">combine</span> them. Does <span class="lemon red">order</span> matter?</p>

The order of the class names in the attribute does not matter, but the order of the styles in the CSS file does matter.

Waterfall

No longer very popular, but it's where most companies have come from.

Long cycle - 6 to 24 months

  1. Research and develop requirements
  2. Design
  3. Code
  4. Test

You have a PLAN

You have well designed and thought out requirements

You have time to breathe during the cycle

Waterfall Cons

Not responsive or adaptive. If you discover an issue, plan can be derailed by months.

Coding and testing are often far apart. You may find major issues that require rework or redesign late in the process.

Slow process. A lot of modern software can't work with a 6-18 month turn around.

Agile

The current popular development method

Work in small units of time (1-3 weeks)

Design, develop, and test within the unit of time.

Only release when software is ready to release.

Planning

Product managers work with customers to understand wants

Product managers write user stories based on customer needs.

These are often called epics (or wedding cakes or hills)

Product managers decide which epic to work on next.

The epic is assigned to a development team.

Middlesex Community College - Tools and Technologies for Tech Writers - Winter 2022

Designing

Sometimes called grooming. Development team (called a scrum team) meets with the Product Manager and designs a solution to the user story.

This may include research spikes where you dedicate time to research how to do something.

Break up the epic into stories â€” smaller, managable tasks.

Assign story points to each story to indicate if it's a small, medium, or large task.

Add all these stories to the backlog.

Middlesex Community College - Tools and Technologies for Tech Writers - Winter 2022

Sprint planning

Based on the design, pick some stories to work on for the sprint.

Move the stories out of the backlog, using story points to identify how many the scrum team can work on.

Go through each story and break it down into specific tasks (Dev, QA, Doc).

Assign estimated time to each task.

If scrum team member has available time, pick up bugs or other technical debt tasks from backlog.

Middlesex Community College - Tools and Technologies for Tech Writers - Winter 2022

The Sprint

Sprints (or iterations) are often only 1 to 3 weeks long.

Dev works on coding the changes.

QA works on test plans.

As Dev completes tasks, QA tests the work.

If QA finds bugs, they create new bug tasks for Dev to fix them.

Doc tries to keep up documenting all the changes, and getting reviews.

Scrums

The daily status meeting. Also called stand-up meeting.

Every day the scrum team meets for 15 minutes and goes around the room:

  • What I did yesterday
  • What I plan to do today
  • Any blockers

If there needs to be additional discussion, it should be broken out into a follow up meeting.

Middlesex Community College - Tools and Technologies for Tech Writers - Winter 2022

Closing the Sprint

Cannot close or finish a sprint until all tasks in the sprint are done.

If you weren't able to finish something, can move it to the next sprint.

Often, closing the sprint is done at the same time you're planning the next sprint.

There should also be a sprint retrospective where you go through what happened to figure out what you're doing right and what you're doing wrong.

HTML

HyperText Markup Language

Or, what makes the Internet go.

While there are dozens of different tools that make it work, every single web page on the internet is rendered as HTML.

Almost no one writes HTML pages any more, but most everything needs to appear as HTML.

Therefore, you need to know how HTML should work.

Types of HTML

There are three major types of HTML around currently

  • HTML 4
  • XHTML
  • HTML5

HTML 4

HTML 4 - First introduced around 1997, it's what most folks think of when you think of HTML. It was not strict, meaning you didn't have to close tags. Easy for amatures to write, but hard for browsers to process. This led to browsers interpreting files in interesting ways.

A proper HTML 4 file starts with a DOCTYPE declaration similar to the following:

<!DOCTYPE HTML PUBLIC
 "-//W3C//DTD HTML 4.01//EN"
 "http://www.w3.org/TR/html4/strict.dtd">

XHTML

XHTML - A version of HTML that is actually valid XML, which is strict. You have to be more exact as you write it, but means browsers can display it regularly.

<!DOCTYPE html PUBLIC
 "-//W3C//DTD XHTML 1.0 Strict//EN"
 "http://www.w3.org/TR/xhtml1/DTD/xhtml1-strict.dtd">

Because these are XML files, you can use XSLT to manipulate the files.

What HTML is

It's a presentation format. It doesn't do anything.

HTML is a language interpreted by a browser to display content.

HTML has hooks in it to other languages that can do stuff, such as JavaScript and Cascading Style Sheets (CSS).

You reference JavaScript and CSS from inside HTML elements, so dividing lines get fuzzy.

Basic markup

HTML is a relative of SGML (Standard Generalized Markup Language) and XML (eXtensible Markup Language)

For this family of markup languages, markup is defined by tags

A tag surrounds content and other tags.

A tag inside of another tag is called nesting. (More on this later)

About tags

Tags are made up of elements

An element is identified by angle brackets < >.

Elements are opened and closed.

To open an element, you use the element name surrounded by angle brackets: <html>

To close an element, you use the element name preceded by a forward slash, surrounded by angle brackets: </html>

More About Elements

All elements should be opened and closed.

Some elements don't need content, and can be opened and closed at once. These are called singletons. A common example is the "break" element.

Attributes

Elements can contain attributes.

An attribute provides extra information that the element might need. For example, the element that defines a graphic minimally needs the path to the image file.

Attributes are name value pairs.

<img src="path/to/image.jpg"></image>

In the above example, the attribute is src

The value is path/to/image.jpg

More About Attributes

Attributes can have multiple values.

The values are usually space delimited, but it can depend on the attribute.

Elements can have multiple attributes.

Required Elements

All HTML files require the following:

A doctype declaration. <!DOCTYPE html>

The root element must be <html>.

The <html> must contain a <head> and <body>.

The <head> contains technical bits that help the HTML work. Minimally it contains a <title>. This is the text that appears in the browser tab.

The <body> contains the elements that display content on the page.

Block vs Inline

There are two major types of content elements, block and inline.

Block elements are discrete things on a page. A paragraph (<p>) is a sample block.

Inline elements are things that fit inside of blocks. Formatting elements, such as <b> for bold are inline elements. Inline elements must be inside of block elements.

To confuse things, you can also nest blocks inside of blocks. For example, having multiple <p> elements in a <li> element.

Styling

You add the @style attribute with CSS property:value; pairs.

<p style="color:blue;">Blue text</p>

However, this means you have to style each and every element you add.

Instead, add a @class attribute with some name.

Then you define the formatting for that class in CSS you reference.

<p class="blue_text">Blue text</p>
Middlesex Community College - Tools and Technologies for Tech Writers - Winter 2022

Styling continued

The CSS.

By class name:

.blue_text {
  color: blue;
}

By element:

p {
  font-size: 10pt;
}

The styles cascade into each other. Which can be fabulous or terrible.

Referencing styles

Besides using @style, you can reference CSS information from the <head>.

You can put the CSS in the HTML file itself in the <style> element in the <head> section.

<head>
    <style>.blue_text {color: blue;}</style>
</head>

Or you can write separate files that contain all the CSS and reference them using a <link> element.

<head>
    <link rel="stylesheet" href="mystyles.css"/>
</head>

div and span

Since most people want to tweak all their formatting themselves, a good portion of HTML is written using <div> and <span>.

  • <div> is for block elements.
  • <span> is for inline elements.

Unless you need the advantages of special elements (e.g. headings), a lot of formatting is done with <div>, <span>, and @class.

Navigating HTML

When working with HTML, you often have to describe where you are in the file.

As mentioned earlier, you can nest elements.

<ul>
    <li>This line item is nested inside the ul element
        <p>This paragraph is nested inside the li element</p>
        <p>There is an <em>inline element</em> nested here</p>
    </li>
</ul>

People often refer to these elements as being in 'a tree'

The path to the <em> element is ul/li/p/em

Element Relatives

You also refer to elements by their relations

<li><p>para nested in li</p></li>

The <li> is the parent of the <p>

The <p> is the child of the <li>

And the analogy continues. You can have grandparents, grandchildren, ancestors, siblings, and so on.

If you want to dig into this, it's called xpath.

Document Object Model

DOM - How to reference elements and attributes in HTML files. Not for raw HTML, but important for JavaScript.

<div class="body">
  <h1>Heading 1</h1>
  <p class="first_para">First paragraph</p>
</div>
  • By element: Follow "tree" and use dot notation: div.p.
  • By class: p.first_para.
  • Depends on how you're trying to find an element or attribute the exact notation.

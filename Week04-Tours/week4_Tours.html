Tour guidelines

  • Shorter is better.

    Think how annoyed you get with a long tour.

    This goes for text in the popups and the number of popups.

  • Often use more informal voice.
  • Beware of tours that require pre-reqs and changing screens.
Middlesex Community College - Tools and Technologies for Tech Writers - Winter 2024

Tour considerations

  • Make a script or a mock up first
  • Consider the purpose of the tour - new feature vs getting started vs common task
  • Is the tour automated? (more like a walk through)
  • Is the tour interactive? (more like a tutorial)

Are there additional features? Starts automatically? Only shows the first 3 logins?

Pros

Tour vs Video

Sometimes, tours or walkthroughs become more complicated, and it becomes better to turn it into a video. Make a video if:

  • need to change screens a lot
  • need data set up in a certain way
  • not able to get a developer to help you make tours work

Wizards

A software wizard or setup assistant or multi-step form is a user interface that leads a user through a sequence of small steps,[1][2] like a dialog box to configure a program for the first time. A complex, rare, or unfamiliar task may be easier with a wizard that breaks the task into simpler pieces.

Tutorial

A step-by-step walk through to produce a specific result.

Many people learn best by doing.

Walking through a specific example can help people learn complicated software.

Related to but different than training.

Tutorials are just a procedure. Use whatever your authoring in.

Middlesex Community College - Tools and Technologies for Tech Writers - Winter 2024

Tutorials continued

Just a procedure, but more specific than a user's guide.

  • Pick something discrete and contained.
  • Provide a lot more context than a general procedure.
  • Explain what you're going to do and how you're going to do it.
  • Walk through the results confirming that you did what you started out to do.
  • Still follow general tech writing best practices, and keep procedures short.
Middlesex Community College - Tools and Technologies for Tech Writers - Winter 2024

Tutorial tips

Talk to your SMEs (Subject Matter Experts) and PMs (Product Managers) (and implementation teams if you have them) and learn your audience.

Try to find the balance between too simplistic "hello world" and too crazy "and magic happens here"

Work with training for good examples.

While it is "just a procedure" you may want slightly different layout.

Middlesex Community College - Tools and Technologies for Tech Writers - Winter 2024

More tutorial tips

Tutorials generally have more graphics. You'll want a lot of before and after comparisons. However, watch out for maintenance.

You might want special tips, important point, checkpoint, sidebar, etc. formatting.

Also consider how the user will be reading the tutorial. Is it printed out? In a slide out in the app? A PDF in a separate window?

Don't forget making sure any required files are included.

Tweaking titles

When defining a title, you can provide <titlealts> and provide

  • <navtitle> — An alternative title to use in any table of contents or navigation
  • <searchtitle> — An alternative title to use in search results

You can provide <titlealts> in topics and in ditamaps.

Conditional text

Include, exclude, and flag text

Almost every element can take one of the profiling attributes:

@audience, @deliveryTarget, @otherprops, @platform, @product, @props

You can define values for these attributes, and then include and exclude content based on the values.

You can also do other processing, such as make all @audience="expert" content appear with a different background color.

There is also the @rev attribute which can be used for flagging, only.

Keys

Keys are a very powerful mechanism in DITA.

A key is an addressing method, defined in ditamaps.

Generally, use keys for a link destination or a short text string.

  • Have a key for "product", so in topics about a shared component, you use the product key.
  • Have a set of keys that link to images. The help map links to low resolution files, the PDF links to high resolution files.

Conref push

Normally, when you use a conref, you replace the current element with the content of some other element.

With conref push, you can insert (or replace) content before or after an element with an ID.

For example, you set up reference topics that always have an <example> section with an @id that equals example_commandname.

In a specific output, you can insert an extra section that's required for an OEM version of the application.

Semantic tagging

Instead of applying styles, take information about what it is.

Instead of worrying about formatting, just tag information for what it is. Let the publication tools worry about formatting.

Avoid using <b> and <i>.

Work with your editors to determine how to tag items using <uicontrol>, <wintitle>, <parmname>, <codeph>, etc.

Conrefs and Keys

The next phase of reuse.

Use the @conref attribute to reuse almost any DITA element.

Conrefs only work with like elements. To reuse a <note>, you have to insert a <note>

Conrefs are associated with files.

Use the @keyref for a key, most commonly used for short words or phrases or paths to files.

Keys are associated with maps.

Middlesex Community College - Tools and Technologies for Tech Writers - Winter 2022

Example conref

Source item

<p id="reuseme">This text is reused</p>

To reuse this paragraph somewhere:

 <p conref="path/to/topic.dita#topic_id/reuseme"/>

Most DITA editors have tools to help you insert the conref.

Remember, you have to use the same element. The following does not work:

 <note conref="path/to/topic.dita#topic_id/reuseme"/>

Conkeyref

Conrefs are fantastic...except you are constantly inserting the path to a DITA topic.

If you have a shared topic used by everyone with your legal warnings or product names, you may have hundreds or thousands of references to the file containing the conrefs.

If you need to rename or move that file, you now have to update hundreds or thousands of references.

Instead, use a conkeyref.

Same as a conref, except you define the path to the file as a key in your map.

<p conkeyref="keyaname/id"/>

Conrefs and conditions

You can use conditions and keys in content you reference.

One conref, many products

<ph id="productName"><ph product="prodA">Product Alpha</ph>
<ph product="prodB">Product Bravo</ph>
<ph product="prodC">Product Charlie</ph></ph>

Then, whenever I want to insert the product name in a topic used in all three products, all I insert is

<ph conkeyref="conreffile/productName"/>

Example key phrase

Keys are defined in maps.

To define a string in a map file:

<keydef keys="key_name">
    <topicmeta>
        <keywords>
            <keyword>Words from a key</keyword>
        </keywords>
    </topicmeta>
</keydef>

Then, in any topic in that map, all you need to insert is

<ph keyref="key_name"/>
or
<keyword keyref="key_name"/>

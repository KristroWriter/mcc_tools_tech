Paths

How we link to things.

While many applications have tools that help, you will need to write paths by hand, and troubleshoot them.

Two major types:

  • Fully Qualified
  • Relative
Middlesex Community College - Tools and Technologies for Tech Writers - Winter 2022

Fully qualified paths

Precise description of the location of a thing (folder or file on a computer).

Longitude and Latitude coordinates are fully qualified paths.

Windows: C:\Users\zlawson\Documents\mcc\example.txt

Unix: /usr/zlawson/mcc/example.txt

It's exactly how to get to the file, but if you change computers, the path most likely will not work.

URLs

FYI, a URL (Uniform Resource Locator) is a fully qualified path.

http://www.example.com/directory/file.html

  • http - the protocol to reach the remote computer
  • www.example.com - the name of remote computer
  • directory/file.html - the path and name of the file to load
Middlesex Community College - Tools and Technologies for Tech Writers - Winter 2022

Relative paths

How to get to someplace else from where you are.

GPS directions are a relative path.

  • file.txt - a file that's in the same directory as the file you're working in.
  • directory/file.txt - a file that's in a folder called directory in the same directory as the file you're working in.
  • ../file.txt - a file that's in the parent folder of the directory of the file you're working in.

Troubleshooting paths

OS specific. Windows uses \, Unix uses /

However, HTML, XML, and other languages require / regardless of OS

Copy and paste into a text editor where you can change the font to something monospace

Don't be afraid to use editor tools to enlarge the font to read them.

Use your finger to read carefully.

Case matters

What is lightweight markup?

When you want more than text but you don't want it to be "complicated" or "heavy".





https://xkcd.com/927/

Pros of lightweight markup

It's lightweight.

Easy to write, just need a text editor

"Anyone" can author in it

Some of the free/low cost editors tout the lightweight part, and how easy it is to just...write. If you need fewer distractions to get text on a page, markdown could be the thing for you.

Middlesex Community College - Tools and Technologies for Tech Writers - Winter 2022

Cons of lightweight markup

It's lightweight.

Good documentation has hard problems to solve (variables, links, tables, images) that are hard to do with lightweight tools

While the formatting is "easy", it's finicky. A lot of it requires the correct amount of white space to be rendered correctly. Without "tagging" can be harder to troubleshoot.

Cons of lightweight markup

Because it doesn't do everything that's needed, it gets extended into different flavors that become complicated (and mimic things that already work in "heavyweight" markup).

Heading syntax

Use # to mark headings.

The number of # equates to the heading level.

Markdown supports 6 levels of headings. (Just like HTML)

There has to be a single space after the last #

# Heading 1
## Heading 2

Inline syntax

Use a single * or _ for italic text

*this* and _this_ will be italic

Use double ** or __ for bold text

**this** and __this__ will be bold

Use a backtick ` for a code sample

`Write_this_filename.txt` without italic text

Use two tilde ~~ for strike through text

~~this~~ will be crossed out

List syntax

Use a single * followed by a space for a bulleted list.

Use 1. followed by a space for a numbered list.

You can nest lists, but you need to get the number of spaces before the next bullet correct.

* list
* list 2
  * sub-list
  
1. step 1
   * sub-list
   * sub-list
1. step 2
   1. sub-step 1
   1. sub-step 2
1. step 3

Image and Link syntax

Put the text in square brackets and the URL/URI in parenthesis.

Image links are preceded by a !

![An image](../graphics/image.png)

[Link text](http://www.example.com)

Images need a bit of text. In the HTML world it's called alt text.

It's a bit of text to display if the image can't be found, or for use with screen readers.

Table syntax

Use pipes | to separate columns and a bunch of hyphens - to indicate the header

Source Control

Also known as Version Control or Revision Control

Middlesex Community College - Tools and Technologies for Tech Writers - Winter 2022

What is it?

Tools that maintain a history of all the changes to files.

Software is made up of filesâdozens, hundreds, thousands. We need some way of keeping track of all the changes that happen.

  • To be able to revert to a last known version
  • To be able to work on several versions at once
  • To share files between people easily
  • CYA and Blame
Middlesex Community College - Tools and Technologies for Tech Writers - Winter 2022

Why do you care?

You are going to have to use some (or several) forms of source control to do your job.

No one ever offers training to the tech writers on how to use it.

It's assumed that developers already know how to use it.

If you don't know how to use it, you're going to lose work, overwrite other people's work, and generally be miserable.

Using some form of source control will be a part of your job.

Middlesex Community College - Tools and Technologies for Tech Writers - Winter 2022

What does it do?

With source control, you can share files with lots of people and recover previous versions.

Once you add a file to source control, it becomes versioned.

You can "easily" switch between previous versions of files.

You can mark a set of files & version to make a snapshot in time. For example, what made up a specific release of a document.

Any one with access to source control can get the files.

Source or version control

  • What developers use
  • Intended for smallish text files
  • Not great for binary files (images, pdfs, etc.)
  • Many, many flavors: cvs, svn, git, Perforce, SourceSafe, Rational Team Concert, etc.
Middlesex Community College - Tools and Technologies for Tech Writers - Winter 2022

Document control

  • What marketing or legal uses
  • Generally intended for binary files
  • SharePoint, Documentum. Some Wikis have rudimentary versions.
Middlesex Community College - Tools and Technologies for Tech Writers - Winter 2022

Content management system (CMS)

  • Do what version control does
  • Maintain relationships between documents
  • Often integrated with some sort of publishing system
  • Often includes a document lifecycle/workflow (reviews, approval, end of life)

Traditional vs Collaborative

(My personal definitions)

  • Traditional
  • Collaborative
Middlesex Community College - Tools and Technologies for Tech Writers - Winter 2022

Traditional

Only one person can edit a file at a time. In general, you lock or check out the file, make your changes, then check the file back in.

  • Pros: Only one person edits a file at a time, fewer conflicts.
  • Cons: Clunky. If someone forgets to unlock a file it's a problem. Not currently popular.

Examples include Perforce and CVS

Middlesex Community College - Tools and Technologies for Tech Writers - Winter 2022

Collaborative

In general, even if people are editing the same file, they're solving different problems and editing different parts of the file. It's generally okay for people to just work and sort through the conflicts.

  • Pros: Easy for people to work together.
  • Cons: Easy for people to overwrite each other's work.

This may also be referred to as "Branch and Merge".

Examples include SVN and Git.

Git

Currently the most popular source control tool.

[Linus] Torvalds sarcastically quipped about the name git (which means unpleasant person in British English slang): "I'm an egotistical bastard, and I name all my projects after myself. First 'Linux', now 'git'." The man page describes Git as "the stupid content tracker". The read-me file of the source code elaborates further:

The name "git" was given by Linus Torvalds when he wrote the very first version. He described the tool as "the stupid content tracker" and the name as (depending on your way):

  • random three-letter combination that is pronounceable, and not actually used by any common UNIX command. The fact that it is a mispronunciation of "get" may or may not be relevant.
  • stupid. contemptible and despicable. simple. Take your pick from the dictionary of slang.
  • "global information tracker": you're in a good mood, and it actually works for you. Angels sing, and a light suddenly fills the room.
  • "goddamn idiotic truckload of sh*t": when it breaks

Source: Wikipedia https://en.wikipedia.org/wiki/Git#History

Repositories

Middlesex Community College - Tools and Technologies for Tech Writers - Winter 2022

Important concepts

  • Repository - The major container for Git content. Contains branches.
  • Origin - The symbolic name for the "original" repository that's considered the 'home' or 'source' of the content.
  • Remote - The symbolic name for the repository that is not on your system. Often the same as Origin.
  • Branch - A set of related files. Usually associated with software version or a fix. In general, all branches in a repository are similar sets of files.
Middlesex Community College - Tools and Technologies for Tech Writers - Winter 2022

Important concepts continued

  • Clone - To copy a repository to your system, maintaining a connection so you can make updates to it.
  • Fork - Copy a repository you don't have write permission to, maintaining a weak connection so you can consume updates and request to make changes to it
  • Commit - Officially make the change in the repository with a comment.
Middlesex Community College - Tools and Technologies for Tech Writers - Winter 2022

More important concepts continued

  • Pull - To get the latest and greatest from the remote repository to your local repository. Or to get files from Branch A to Branch B.
  • Fetch - Similar to pull, it gets files from another repository or branch to your local system. In fact, a pull includes a fetch. It does not automatically merge files, but it also gets additional metadata.
  • Push - To get the latest and greatest from your local branch to remote branch.
  • Pull request - Ask someone with permission to grab your changes and pull them into a restricted branch/repository.
